## STUB-002：客服团队页（MembersPage）

- **估计耗时**: 60 分钟
- **目标行数**: ~300 行

### 可用 API

| 端点 | 说明 |
|------|------|
| `GET /api/runtime/agents?account_id=xxx` | 坐席列表（DB） |
| `GET /api/runtime/agents/workloads?account_id=xxx` | 坐席工作负载 |
| `GET /api/agents/presence?account_id=xxx` | 在线坐席（Redis） |
| `POST /api/agents/presence/online` | 标记在线 |
| `POST /api/agents/presence/offline` | 标记离线 |
| `POST /api/agents/{id}/status` | 更新状态 |

### 页面设计

```
┌─ 客服团队 ──────────────────────────────────────────────────┐
│ 统计: 总坐席 5 | 在线 3 | 忙碌 1 | 离线 1                    │
│ [账号选择: 全部 ▾]                                           │
│                                                              │
│ ┌─ 坐席列表 ──────────────────────────────────────────────┐ │
│ │ 坐席名     │ 状态  │ 账号      │ 当前会话 │ 总会话 │ 操作      │ │
│ │ agent-001 │ 🟢在线│ acct-01  │    3    │   15  │ [忙碌][下线]│ │
│ │ agent-002 │ 🟡忙碌│ acct-01  │    5    │   22  │ [在线][下线]│ │
│ │ agent-003 │ ⚫离线│ -        │    0    │    0  │ [上线]      │ │
│ └──────────────────────────────────────────────────────────┘ │
│                                                              │
│ [+ 添加坐席]                                                 │
└──────────────────────────────────────────────────────────────┘
```

### 功能要点

1. 统计栏: 总数/在线/忙碌/离线 计数
2. 坐席列表: 合并 agents(DB) + presence(Redis) + workloads 三个数据源
3. 状态列: 🟢在线 / 🟡忙碌 / 🔴离开 / ⚫离线
4. 操作列: 根据当前状态显示可用操作（上线/下线/切换忙碌）
5. 添加坐席: Modal 表单（agent_id, display_name, account_id）
6. 数据合并逻辑: agents 提供基础信息, presence 提供实时状态, workloads 提供会话数
