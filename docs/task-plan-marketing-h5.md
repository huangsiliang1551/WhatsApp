
### 商品状态说明

| 状态 | 显示 | 可操作 |
|------|------|--------|
| pending (未开始) | ⬜ 待开始 | 前置商品未完成时禁用 |
| available (可开始) | ▶ 开始任务 | 可点击 |
| running (进行中) | 🔄 进度弹窗 | 自动执行中 |
| completed (已完成) | ✅ 完成 | 不可操作 |
| failed (失败) | ❌ 余额不足 [充值] [重试] | 可充值后重试 |

### "开始任务" 进度弹窗

```
┌─ 正在处理 ──────────────────────────┐
│                                      │
│ ✅ 正在下单                          │
│ 🔄 正在支付 (扣除 ¥8.00)           │
│    ━━━━━━━━━━━━░░░░ 75%            │
│ ⬜ 支付完成                          │
│ ⬜ 核对任务状态                      │
│                                      │
└──────────────────────────────────────┘
```

**流程**: 点击"开始任务" → 弹出进度弹窗 → 前端依次调用 API:

1. `POST /api/task-instances/{id}/start-product` body: `{product_id: "xxx"}`
2. 后端处理: 扣系统余额 → 返回成功/失败
3. 前端动画: 模拟 4 步进度（每步 800ms 动画）
4. 成功 → 按钮变为 ✅ 完成 → 下一个商品变为可用
5. 失败(余额不足) → 弹窗提示 "余额不足，当前余额 ¥X.XX" → [去充值] [取消]

### 余额不足提示弹窗

```
┌─ 余额不足 ──────────────────────────┐
│                                      │
│ ⚠️ 系统余额不足                      │
│ 需要: ¥8.00                          │
│ 当前: ¥3.50                          │
│                                      │
│ [去充值]    [取消]                   │
└──────────────────────────────────────┘
```

### 全部商品完成 → 包完成展示

```
┌─ 🎉 恭喜！任务包已完成 ─────────────┐
│                                      │
│ 全部 5 个商品任务已完成              │
│ 奖励 ¥5.00 已发放到任务余额          │
│                                      │
│ [查看余额]    [返回任务列表]         │
└──────────────────────────────────────┘
```

### 串行逻辑

- 商品按列表顺序排列
- 只有当前商品状态为 completed 或 failed 时，下一个商品才变为 available
- 失败的商品可以重试（充值后点"重试"按钮）
- 重试时调用 `POST /api/task-instances/{id}/retry-product`

### API 调用

| 操作 | API |
|------|-----|
| 加载包详情 | `GET /api/task-instances/{id}` |
| 开始商品任务 | `POST /api/task-instances/{id}/start-product` |
| 重试失败任务 | `POST /api/task-instances/{id}/retry-product` |
| 充值跳转 | 跳转 `/recharge` 页面 |

## 四、ProfilePage 签到按钮（+50行）

### 修改位置

在现有 ProfilePage 的手机号显示区域右侧增加签到按钮：

```
┌─ 个人信息 ────────────────────────────┐
│ 手机号: 138****1234   [🔥 签到]       │
│ (或) 已签到: ✅ 连续 5 天             │
└────────────────────────────────────────┘
```

### 逻辑

1. 从 `/api/sign-in/status` 获取签到状态
2. 如果签到任务已完成（`is_completed: true`），不显示签到按钮
3. 如果今日已签到，显示"✅ 连续 N 天"
4. 如果今日未签到，显示"🔥 签到"按钮 → 点击调用 `POST /api/sign-in`

### 签到成功动效

```
签到成功！🔥 连续 6 天
距奖励还差 1 天
```
