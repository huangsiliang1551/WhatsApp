#### 数据合并逻辑

前端需合并 3 个数据源：

| 数据源 | API | 提供字段 |
|--------|-----|---------|
| DB 坐席列表 | `GET /api/runtime/agents` | agent_id, display_name, email, account_id, status |
| Redis 在线状态 | `GET /api/agents/presence` | agent_id, status(实时), last_heartbeat |
| 工作负载 | `GET /api/runtime/agents/workloads` | agent_id, assigned_open_conversations, assigned_total_conversations |

合并规则：以 DB 坐席为基础，用 agent_id 关联合并 Redis 在线状态和工作负载。

#### 操作按钮逻辑

| 当前状态 | 可用操作 | 调用 API |
|---------|---------|---------|
| 🟢在线 | [忙碌] [下线] | `POST /api/agents/{id}/status` |
| 🟡忙碌 | [在线] [下线] | `POST /api/agents/{id}/status` |
| 🔴离开 | [在线] [下线] | `POST /api/agents/{id}/status` |
| ⚫离线 | [上线] | `POST /api/agents/presence/online` |

#### 功能要点

1. **三数据源合并**: agents(DB) + presence(Redis) + workloads
2. **实时状态**: 优先取 Redis presence 状态，无则取 DB status
3. **状态操作**: Popconfirm 确认后调用对应 API
4. **注册坐席**: Modal 表单 + `POST /api/runtime/agents`
5. **自动刷新**: 30 秒自动刷新在线状态
